# 🚀 SPACE DASH

> 터미널에서 즐기는 ASCII 기반 레이싱/생존 게임

---

## 📌 게임 소개

**SPACE DASH**는 콘솔(터미널) 환경에서 실행되는 ASCII 아트 기반 레이싱 게임입니다.  
트랙이 위로 스크롤되는 동안 플레이어는 좌우로 이동하며 장애물을 피하고, GOAL 라인 도달 시 클리어됩니다.

---

## 🎮 게임 모드

### NORMAL 모드
- 벽에 닿아도 게임이 끝나지 않고 차량 위치가 도로 안쪽으로 보정(Clamp)
- 고정 장애물(바위/콘)과 충돌 시 GAME OVER
- 아이템 점수 시스템 적용 (★ +5 / ● +3 / ▲ -2)
- 15초, 30초 구간마다 게임 속도 증가
- 최고점수 저장 (`highscore_normal.txt`)

### HARD 모드
- 벽, 장애물, 미사일 모두 즉시 GAME OVER
- **미사일 시스템**: 화면 측면에서 `<====` 형태의 미사일이 주기적으로 생성 (초반 3초 안전시간 존재)
- **미니맵(HUD)**: 화면 우측에 전체 트랙 진행도 및 플레이어 위치 표시
- 플레이 시간에 따라 게임 속도 점진적 증가
- 최고점수 저장 (`highscore_points.txt`)

## 🕹️ 조작법

| 키 | 동작 |
|---|---|
| `A` / `←` | 좌측 이동 |
| `D` / `→` | 우측 이동 |
| `ESC` | 게임 종료 |

---

## 🗺️ 오브젝트 가이드

| 문자 | 오브젝트 | 설명 |
|---|---|---|
| `│` | 도로 경계(벽) | 이동 불가 영역 |
| `.-.` / `\|_\|` | 차량 | 플레이어 차량 (3×4 ASCII Art) |
| `★` | 아이템 (Star) | 획득 시 점수 +5 |
| `●` | 아이템 (Circle) | 획득 시 점수 +3 |
| `▲` | 아이템 (Triangle) | 획득 시 점수 -2 (감점) |
| `<====` | 미사일 | HARD 전용, 충돌 시 즉사 |
| `^` / `/ \` | 장애물 | 바위/콘, 충돌 시 게임 오버 |

## 🛠️ 사용 기술

- **언어**: Python 3
- **핵심 모듈**: `time`, `shutil`, `unicodedata`, `keyboard`
- **화면 처리**: ANSI 커서 이동 방식으로 깜빡임 없는 렌더링
- **충돌 판정**: `set` 자료형 기반 좌표 셀 교집합
- **맵 구조**: 파일 기반 ASCII 트랙 (`track.txt`, `track_hard.txt`)

### 🔍 한수창 상세 기여

**HARD 모드 게임 시스템 (`hard.py`)**
- Hard Mode 전용 게임 루프 설계 및 구현
- `spawn_missile()` — 일정 간격 미사일 생성
- `move_missiles()` — 미사일 이동 및 화면 밖 자동 제거
- `check_missile_collision()` — 플레이어/미사일 좌표를 `set`으로 변환 후 교집합으로 충돌 판정, 충돌 시 즉시 GAME OVER
- `update_speed_by_time()` — 플레이 시간에 따른 점진적 속도 증가

**트랙 맵 설계 및 구현**
- NORMAL / HARD 모드용 트랙 데이터 직접 설계 (`track.txt`, `track_hard.txt`)
- 직선·대각선 구간이 자연스럽게 이어지는 레이아웃 구성
- 플레이 가능 영역(벽/도로)을 명확히 구분하여 안정적인 주행 환경 구현

**Normal 모드 이동 오류 수정 (`normal.py`)**
- 대각선 트랙 구간 진입 시 발생하던 벽 이탈 오류 분석 및 수정
- 이동 가능 범위 계산 로직 재설정으로 위치 보정 현상 제거

**메인 UI (`main.py`)**
- `draw_menu()` — Normal / Hard 모드 선택 화면 구현 (↑↓ / Space 입력)
- `screen_one()` — 타이틀 출력 및 모드 선택 루프 제어
- `screen_three(score)` — 게임 종료 후 결과 화면 및 재시작 분기 처리
- `build_info_box_left()` — 터미널 폭에 따른 동적 UI 박스 생성
